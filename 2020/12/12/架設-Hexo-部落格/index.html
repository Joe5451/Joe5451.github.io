架設 Hexo 部落格


什麼是 Hexo

Hexo 是一個能夠讓使用者快速建立部落格網誌的框架，使用 Markdown 標記語言解析文章，如果編寫過專案的 readme.md 或是使用過 HackMD 撰寫過筆記應該會比較熟悉 Markdown，第一次接觸或是不熟悉的可以在網路上搜索，但本文還不會涉及到該部分。

安裝需求

在安裝 Hexo 之前，電腦必須安裝 Node.js 以及 Git，關於安裝這兩者的細節這裡就不多贅述，網路上已有很多教學，但要注意官方文件標明 Node.js 版本需不低於8.10，建議使用 10.0 以上版本。

此外使用 Hexo 框架架設部落格，最後會佈署到 GitHub 上，所以如果沒有 Github 帳戶的使用者也要申辦一個。

配置

開啟專案中的 _config.yml 檔案，可以設定一些屬性，如網站的標題、描述、作者的名字…等，更換網誌主題及佈署的配置也要在這裡做設定，這些部分在後面用到時再提。


更換主題

不喜歡 Hexo 預設的主題，可以到 Hexo 官方的 Themes 尋找中意的主題，或者你也可以參考官方的文件自己做一個主題，但筆者沒有學習這部分，所以就從中挑選一個使用。

佈署到 GitHub

基本的部落格已成形，但只能在本地端的電腦預覽，想要讓其他人也能看到則必須要佈署到一個平台上，平台的選擇有很多，例如 GitHub、Heroku 等，這裡將採用 GitHub 的方式，因此沒有 GitHub 帳戶的請先申辦一個。

登入 GitHub 後，在右上角選擇 New repository 新增儲存庫


專案名稱取為 username.github.io，注意 username 的部分要與 Owner 一樣，例如 Owner 為 user123 則專案名稱必須取為 user123.github.io

更新部落格

每當新增文章時，要先輸入 hexo clean 指令消除快取檔案 (db.json) 和已產生的靜態檔案 (public)，清理完後再 hexo generate 及 hexo deploy 佈署。

題外

GitHub 改變 default branch 預設分支

到 Settings 選擇 Branches

切換想要作為預設分支的 branch，並按下 Update 即可

GitHub 刪除分支

在專案點選分支，點擊 View all branches

點擊 All branches 對想要刪除的分支按下垃圾桶的符號即可刪除該分支
